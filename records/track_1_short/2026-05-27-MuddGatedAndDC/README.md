# MUDD-gated & Lightweight DC

This record combines two active model changes on top of the `2026-05-07_XSAGatedLayers` short-track baseline:

1. **DC on the final attention layer.** Layer 10 gets a lightweight post-only, no-DD dynamic correction after the base FA3 attention output. The implementation lives in `dc_triton_kernels.py` and is logged with the run.
2. **MUDD gate.** Two small MUDD-style gating MLPs generate per-token coefficients for selected XSA gates, attention gates, x0/bigram injections, and the layer-3 to layer-6 skip.

The forward also removes or bypasses several redundant trick paths from the previous baseline. The main practical result is that the schedule can be cut from `1375 + 10 = 1385` total steps to `1315 + 20 = 1335` total steps while improving the mean validation loss.

```text

xsa-baseline-1335     9   1335    78.5576   0.0401     -2.6611     3.2848  0.0017  1.00e+00
this_pr-s1335        11   1335    78.9669   0.0425     -2.2518     3.2771  0.0012  8.15e-06
```

`Time delta` is relative to `xsa-baseline-s1385`. `Loss p` is the one-sample t-test against the 3.28 target:
`scipy.stats.ttest_1samp(losses, 3.28, alternative="less").pvalue`.

The final `s1315e20` PR sweep passes the `p < 0.01` rule with `p = 8.15e-06`. Compared with the valid `xsa-baseline-s1385`, wall-clock is faster by **2.25s** on the same 8xH100 machine; Welch one-sided t-test for `this_pr < baseline` gives `p = 5.26e-22`. The mean loss is also lower than that baseline by `-0.00127` (two-sided Welch `p = 0.035`).

The new `xsa-baseline-1335` logs are the matched-step comparison for the PR schedule. At `1315 + 20 = 1335` steps, the XSA baseline has mean loss `3.28477` and does not pass the target (`p = 0.99999`), while this PR has mean loss `3.27714`. The matched-step gap is therefore about **0.00763 loss** (Welch two-sided `p = 1.44e-08`). A DC-only run accounts for about **0.0046** of that gap, leaving about **0.0031** attributable to the MUDD-gate / cleanup part of the change. In other words, DC is the main quality contributor, but the MUDD-gated replacement and redundancy cleanup are also doing measurable work.

## DC change

The DC path is deliberately narrow:

- `dc_layers = [10]`, so only the final non-paired attention layer uses it.
- `dc_gate(x, dc_w, num_heads)` computes a small gate path (`dc_hidden_dim = 2 * num_heads = 12`) and emits two per-token, per-head post weights.
- `post_w1` is RMS-normalized across heads before entering the correction kernel.
- The base attention still comes from `flash_attn_varlen_func`; the Triton DC kernel adds the post-only correction to that base output.
- The submitted kernel is the trimmed path for `H=6`, `D=128`, window `112`, and no dense-dense term.

The idea is inspired by [Improving Transformers with Dynamically Composable Multi-Head Attention](https://arxiv.org/abs/2405.08553). The paper's full DCMHA uses both pre- and post-composition and dynamically generates three terms: `w1`, `w2`, and `dd`. This PR intentionally uses a smaller variant: post-composition only, no pre-composition, and no `dd`. The reason is overhead. Full DC is more expressive, but too expensive for this short-track setting unless it is fused much more tightly.

Formula-wise, DC is a dynamic correction on top of the normal attention probabilities. Let `P` be the attention probability matrix and `V` the value tensor. The base attention output is

```text
attn_out = P * V
```

DC predicts two input-dependent weights from the attention input:

```text
W1, W2 in R^{B x T x N}
```

where `B` is batch, `T` is token position, and `N` is the number of heads. These weights are dynamic: they are generated by a small MLP from the same hidden state that feeds attention. The output can be read schematically as

```text
out = P * V + P * W1 * W2 * V
    = P * (1 + W1 * W2) * V
```

The first term is the normal attention output; the second term is the DC correction. In other words, DC makes the attention probabilities dynamically reweighted before they are applied to `V`, without changing the QKV/O projections.

Known isolated experiments outside this folder suggest DC alone is worth about **-0.0045 validation loss**. The runs here do not include a DC-only ablation, but this is currently the best-understood source of loss headroom in the combined change.

In principle, this correction can be fused into FlashAttention by folding the dynamic reweighting into the probability/value accumulation. This submission keeps it separate: FA3 computes the base attention, then the custom Triton kernel recomputes the layer-10 QK softmax over the local window and applies the DC correction. That split is simple and keeps the experiment isolated, but the custom Triton path is much slower than FA3. At matched 1335 steps, the PR step average is `59.15ms` vs `58.84ms` for `xsa-baseline-1335`, about **+0.31ms/step**, and the total matched-step wall-clock cost is **+0.41s**. Against the valid 1385-step baseline, the 50-step schedule cut still wins by **2.25s** overall. This is also good news: DC appears to buy real loss headroom even with nontrivial overhead, so a future FA3-fused implementation should expose more of the quality gain as wall-clock speedup.

## MUDD gate change

The earlier MUDD connections from `2026-04-22_MuddFormer` remain: layer 9 produces last-layer V/residual/dynamic-lambda coefficients, and a post-loop MUDD block mixes `{cache[0], cache[7], cache[9], ve_bank0, cache[3]}` before the final norm.

This record adds a separate MUDD gate stack:

- `mudd_gate_w1`, `mudd_gate_w2`, `mudd_gate_b2`, with hidden dim `64` and scale `0.1`.
- A pre-layer gate from `x0` generates XSA gates for layers `1, 3, 4`, attention gate coefficients for layer `3`, and x0/bigram injection gates for layers `0..5`.
- A post-skip gate generated at layer `6` produces the layer `7` XSA gate, layer `10` attention gate, x0/bigram injection gates for layers `7..10`, and the coefficient for the layer-3 skip.
- Bias init matches the old fixed behavior where possible: x0 starts at `0`, bigram starts at `0.05`, attention gate bias starts at `0.5`, and the skip lane starts at `1.0` before the global `sigmoid(skip_lambda)` multiplier.
- MUDD gate parameters use replicated Adam with `betas=(0.9, 0.99)`, `lr_mul=0.15`, and no weight decay on the bias.

There is not yet a pure MUDD-gate-only ablation. The matched-step comparison above suggests the MUDD-gate / cleanup portion contributes about **0.0031 loss** beyond DC alone, but this is a combined number for the gate plus surrounding cleanup, not an isolated MUDD-gate-only effect.

## Cleanup

The new forward path removes or bypasses a few older tricks that looked redundant in this setting:

- Static per-layer XSA alphas are no longer directly consumed for `{1, 3, 4, 7, 8, 10}`. The active XSA gates are MUDD-generated for `{1, 3, 4, 7}`, which also drops the old layer `8` and `10` XSA use.
- The broad `attn_gate_bank -> sigmoid(F.linear(x[:12], w))` path is bypassed in favor of sparse MUDD-generated attention gates.
- Fixed `x0_lambdas` and `bigram_lambdas` are no longer the active injection path; x0/bigram injection is generated per token by the MUDD gate.
- The old standalone `skip_gate(x0)` path is replaced by a MUDD-generated skip coefficient at the skipped layer.

The reason this cleanup is plausible is that DC overlaps conceptually with parts of XSA and `attn_gate_w`: all three are trying to modulate attention output based on information already present in the attention stream. Once DC is active on the final layer, some XSA / attention-gate layers become partly redundant. Removing those redundant paths contributes to the time gain and keeps the added DC cost from dominating the run.

Some old parameter slots are still present in the file because this was an experiment-driven cleanup, not a full minimal-code rewrite.
